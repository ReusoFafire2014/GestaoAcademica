- O que é reuso de software? 
=> É poder incorporar a novas soluções algo já implementado antes, poupando assim esforço e custo. Por exemplo reuso de classes e métodos para desenvolver aplicações ou até mesmo componentes e frameworks.

- Quais os benefícios esperados do reuso de software?
=> Maior velocidade para implementação, mais facilidade para manutenção de códigos fontes, centralização de regras de negócio e menor impacto ao aplicar mudanças ao sistema.

- Diferencie Reuso Ad Hoc de Reuso Sistemático
=> Ad HOC Parte de iniciativas individuais e não tem nenhuma metodologia definida, enquanto sistemica tem total apoio organizacional e tem um processo a ser seguido para implementá-lo.

- Diferencie componentes de software e frameworks
=> Frameworks são desenvolvidos para solucionar problemas expecíficos que são comuns várias pessoas, eles implementam a solução e difundem entre a comunidade, uma de suas desvantagens é que o sistema fica fortemente acoplado a ele criando uma dependencia e a curva de aprendizagem necessária para começar a trabalhar com ele. Componentes são expecíficos do próprio sistema que serão utilizados diversas vezes mas apenas no mesmo sistema.
